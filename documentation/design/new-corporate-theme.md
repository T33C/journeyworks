Colour sequence
When presenting similar data, it is advisable to use colours close to each other on the colour spectrum, such as blue, purple and pink. For contrasting data, such as loss and gain, opposing colours are recommended.
Please note: In digital environments, data visualisation colours can be used on either a white or black background. However, for print, data visualisations should only be displayed on a white background to meet the 3:1 colour contrast ratio requirement.
To make sure that all accessibility criteria are met, please follow the correct labelling as set out in the Data visualisation standard

RAG palette
The RAG palette (Red - Amber - Green) uses the traffic light system to convey status or severity levels within data visualisation and messaging
For this reason, the Core palette should not be used on app or in web charts and graphs.
• ﻿﻿Red: a strong negative sentiment - errors or unsuccessful outcomes.
• ﻿﻿Amber: a negative sentiment - an alert or warning.
• ﻿﻿Green: a positive sentiment or confirmation.Blue: useful information; usually no action required.
